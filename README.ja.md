# exam05 - 42 Tokyo 練習プロジェクト集

42TokyoのExam05対策

## プロジェクト構成

### レベル1：データ構造とオブジェクト指向プログラミング（C++）

#### 1. **bigint** - 任意精度整数演算
任意の精度を持つ符号なし整数を文字列ベースのストレージで扱う`bigint`クラスの実装。
- **主な機能：**
  - 加算演算のサポート
  - 比較演算子
  - 桁シフト演算（左シフト：10^nを乗じる、右シフト：10^nで除算）
  - `<<`演算子を使用したストリーム出力（先頭ゼロなし）
- **ファイル：** `bigint.hpp`, `bigint.cpp`

#### 2. **polyset** - SetとBagデータ構造
複数のベースとなるデータ構造を使用した検索可能なBagおよびSetクラスの実装。
- **構成要素：**
  - `bag`: Bagデータ構造の抽象基底クラス
  - `searchable_bag`: 検索機能を追加する抽象インターフェース
  - `array_bag`: 配列を使用したBagの実装
  - `tree_bag`: 二分探索木を使用したBagの実装
  - `searchable_array_bag`: array_bagの検索可能版
  - `searchable_tree_bag`: tree_bagの検索可能版
  - `set`: 検索可能なBagをSetに変換するラッパークラス
- **すべてのクラスはカノニカル形式に従う**
- **ファイル：** 仕様書に記載された複数の.hppおよび.cppファイル

#### 3. **vect2** - 2次元ベクトル数学
2次元整数ベクトルの線形代数演算を扱うベクトルクラス。
- **機能：**
  - ベクトルの加算
  - ベクトルの減算
  - スカラー倍
  - `[]`演算子を用いたコンポーネントアクセス（インデックス0と1）
  - ストリーム出力フォーマット：`{x, y}`
- **ファイル：** `vect2.hpp`, `vect2.cpp`

### レベル2：アルゴリズム実装（C言語）

#### 4. **bsq** - 最大正方形探索
2次元マップから障害物を避けて最大の正方形を見つけるプログラム。動的プログラミングを使用。
- **入力形式：**
  - 最初の行：`<行数> <空き文字> <障害物文字> <塗りつぶし文字>`
  - 空きスペースと障害物を含むマップ
- **アルゴリズム：** 最大正方形サイズを計算するための動的プログラミング
- **出力：** 元のマップの最大有効正方形を塗りつぶし文字で置き換えたもの
- **同点時の判定：** 複数の解が存在する場合、最上部、次に最左端の正方形を選択
- **エラー処理：** マップの整合性を検証し、無効なマップに対して"map error"を出力
- **入力方式：** ファイル引数または標準入力
- **ファイル：** `*.c`, `*.h`

#### 5. **life** - コンウェイのライフゲーム
カスタマイズ可能なボード上でコンウェイのライフゲームをシミュレートします。
- **使用法：** `./life 幅 高さ イテレーション数`
- **描画コマンド：**
  - `w` / `a` / `s` / `d`: ペンを上/左/下/右に移動
  - `x`: ペンの切り上げ/下ろし（描画/消去の切り替え）
- **シミュレーション：** 指定されたイテレーション数の間、ライフゲームの古典的なルールを適用
- **出力：** 最終的なボード状態（生きたセル：`0`、死んだセル：スペース）
- **境界：** ボード外のセルは死んでいると判定される
- **ファイル：** `*.c`, `*.h`

## コンパイル方法

各プロジェクトに含まれる`main.c`または`main.cpp`が実装したクラス/関数と共にコンパイルされるようになっています。

```bash
# C++プロジェクトの例
g++ -std=c++11 -Wall -Wextra -Werror *.cpp -o program

# Cプロジェクトの例
gcc -std=c99 -Wall -Wextra -Werror *.c -o program
```
